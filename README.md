# Student Performance Dashboard

A Java Swing dashboard that turns a simple CSV of student marks into a visual analytics workspace. It calculates class-level summaries, lets you explore individual performance, and offers charts, statistics, data quality checks, and export options.

## What this project does
- Loads student marks from student_marks.csv.
- Computes class averages, pass rate, topper, ranks, and percentiles.
- Shows per-student charts and a sortable table of all students.
- Provides analytics charts (distribution, grade bands, correlation).
- Includes descriptive statistics (mean, median, variance, std dev, IQR).
- Flags data quality issues (missing names, duplicates, invalid scores).
- Generates insights and supports chart/table exports.

## How to run
From the project root:

```
javac -cp "lib/*" src\*.java
java -cp "lib/*;src" Main
```

## How it runs (step by step)
1) Data loading
  - Main reads student_marks.csv line by line.
  - The header row is skipped.
  - Each row is split by commas into name, maths, science, english.
  - A Student object is created for each row and stored in a list.

2) Data validation and cleaning
  - Scores are expected to be in the range 0 to 100.
  - Missing names and duplicate names are detected.
  - Invalid score values are counted in the data quality panel.

3) Core aggregation
  - Class averages per subject and total are computed.
  - Pass rate is computed using total >= 120.
  - Topper is the student with the highest total.

4) Analytics engine
  - Descriptive stats are computed per subject and total (mean, median,
    variance, std dev, min, max, IQR).
  - Ranks and percentiles are computed for each student.
  - Grade bands are counted (A/B/C/D/F).
  - Correlation is computed for a subject pair in the correlation tab.

5) Dashboard rendering
  - DashboardUI builds tabs and panels on the Swing event thread.
  - Charts are created through ChartUtil and embedded in ChartPanels.
  - The student tab binds the selector to update the per-student chart.

6) Exports and what-if analysis
  - Chart exports are saved as PNG via a save dialog.
  - The student table can be exported to CSV.
  - The what-if slider simulates a subject score shift and updates
    the class average and pass rate.

## Project structure

- student_marks.csv
  - Input dataset with columns: name, maths, science, english.

- lib/
  - Third-party libraries (JFreeChart and JCommon).

- src/Main.java
  - Entry point. Reads the CSV, computes quick summaries, and launches the UI.

- src/DashboardUI.java
  - Main Swing UI. Builds all tabs, charts, tables, cards, and export actions.

- src/ChartUtil.java
  - Chart builder utilities for averages, per-student views, rankings,
    distributions, grade bands, and correlation plots. Also supports export to PNG.

- src/AnalyticsUtil.java
  - Data analytics engine. Calculates statistics, ranks, percentiles, correlation,
    grade bands, and data quality metrics.

- src/Student.java
  - Simple data model for a student. Includes helper methods for totals, average,
    and grade.

- out/
  - Build output directory (generated by some IDEs).

- .vscode/
  - Editor settings (optional).

## Notes
- Pass threshold is set to total >= 120 (out of 300).
- Grades are based on average score (A/B/C/D/F).
- Exports are saved to a location you choose from the Save dialog.
